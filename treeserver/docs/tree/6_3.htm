网络计算方式信息系统特点

浏览器/服务器方式信息系统具有如下特点：

1.图形用户接口

尽管WWW协议本身并不是一个用户接口，现在的大多数浏览器都支持图形用户接口。浏览器提供许多通用的图形元素，比如弹出式菜单，滚动条和复选框等等，以便系统开发者创建用户友好,更易于使用的Web页面。从前面所给的页面示例也可以看出，Web页面具有与一般窗口程序相一致的外观和感受。

2.平台独立性

在世界上存在着许多种类的计算机系统和操作系统，一种系统的软件和数据往往不能用于另一种系统。在WWW出现以前，这是一个相当棘手的问题，软件公司和用户往往需要花费大量的人力和物力来进行不同系统之间的软件移植和资源共享。在WWW出现以后，这个问题已经得到了部分的解决。尽管WWW应用存在一定的局限性，WWW浏览器本身并不一定是平台独立的，但它为用户提供了一个平台独立的用户接口，这样，也就可以开发出平台独立的Web应用来。目前，WWW浏览器几乎可以为任何一种计算机平台，其中有：

Macintosh

基于DOS的PC

基于WINDOWS95/NT的PC

Silicon Graphics ( IPIX )

Hewleff-Packard ( HP-UX )

Sun Microsystems ( Solaris )

Digital Equipment Corporation ( OSF and Ultrix )

NeXT( NeXTSTEP )

IBM RISC System/6000 ( AIX )

通过开发Web方式的信息系统，无需针对不同的系统进行编程就可以支持互不兼容的多平台。与每一种系统都需要单独版本的应用相比，这样的应用更易于维护，只需要较低的维护费用。这也是进行Web方式信息系统开发的原因之一。

3.标准接口

用WWW作为数据库接口的另一优点是一致性。WWW标准是严格的，它提供了一个稳定的平台而不用考虑计算机平台。无论用户何时何地，在何种计算机平台上使用信息系统，都只需要学习一种接口；对系统开发者来说，不仅只需要针对一种接口进行编程，同时也节省了在培训和文档方面的开支。

4.易于扩展

可扩展性是Web标准和开放式接口的另一个优点。假设已有一个用WWW和FOXPRO数据库开发的Web方式的信息系统，如果想要改用TreeSvr数据库，无需重写整个系统，只需重写一个访问TreeSvr的CGI接口就可以了。这样，也就为信息系统开发提供了一个稳定，可靠，用户友好且易于维护的信息系统开发环境。
