前言

      树有树冠、树干、树根三部分，这三部分互为服务，树冠进行光合作用，利用光能和树根从土壤中吸取的养料作为动力，吸收二氧化碳，释放氧气。树干则是一棵树的主干，它撑起一棵树，使它可以傲立于大自然。树根则从土壤中吸取养料。……。多么和谐的整体。自然给我们以启迪。
       树是绿色的。
       我们要设计象树一样的信息系统。
        我们拿树来形象说明一种复杂的数据结构，这种数据结构，“养育”了信息查询这一应用，使计算机比人的本领大了。
       随着信息系统规模的扩大，应用模型越来越复杂。信息的存储、管理、从即有信息资源中获取管理决策信息、利用中央信息系统的能力实现复杂的业务模型等构成了一个信息系统的主要业务，而如何协调客户/服务器方式信息系统的业务传递、信息共享机制更是得以深层次开发。
       在客户/服务器方式信息系统中，信息系统的开发者面对的主要总是是：

  • 如何简化客户端对服务器的访问界面；
  • 如何划分客户业务、还是服务器业务；
  • 划分给服务器的业务如何协调。

  TreeSVR就是一个试图全方位解决这些问题的一个产品，它：

  • 利用事务语言ASQL统一了事务定义；
  • 提供了一组访问共享数据资源的标准API，简化了客户端对服务器数据的访问；
  • 将服务器业务用引擎方式实现，由TreeSVR负责调这些引擎解决服务的并发执行。

     TreeSVR是基于事务的住处系统的一个重要产品，它的最初原型设计开始于1996年底，1997年开发出了第1版，并在医院信息系统这样的大型业务系统中得到应用。在总结第1版的经验教训及应用情况的基础上，开发出了第2版，目前可交付使用的就是这个版本。需指出的是作为TreeSVR的构件的很多部分都是在使用了近十年的软件之上的提高与集成，可靠性都已得到充分证实。
