基础

       强大的计算机处理能力。信息系统能把孤立的、零碎的信息变成比较完整的，有组织 信息系统是指对信息进行收集、储存、传递、加工、维护和使用的系统，它的核心是的信息，提高了信息的效能。一个好的信息系统不只对信息做表面的组织和管理，而且能把信息的内在规律更深在挖掘出来。

        信息系统从人的日常业务、生产决策、知识积累与获取等几个角度为人类提供服务。通常的信息系统都是由以下几个部分组成：

  1. 数据库系统(DataBase System,DBS)
       
        完成系统表示数据的存储、共享、数据级加工等工作。是一个最基本的部件。一般情况下，任何一个信息系统都不可能缺少数据库及其管理系统。但是在一些大型信息中，采用的是集总数据库系统(Intergreating DB System)，以达到资源共享的目的，所以局部信息部件可能不再有自己的数据库系统，这样数据通信和分布共享将成为十分突出的问题。

        数据库系统不只是对原始资料的惧、整理和组织，还能对综合性数据或经过预处理后的数据进行管理。

  2. 事务调度与管理系统(Transaction System,TS)

        事务调度与管理系统（简称事务系统）综合了数据库和程序库，它是一个面向多种应用的程序包，它们是肯有某一特定领域的功能程序，用以描述数据结构和功能要求的通用格式化的接口，通过内部的数据管理系统处理存储的数据。多个大型信息系统的开发与实施的实践表明，数据库系统与事系统的合作程度直接关系到系统的开发组织、开发周期和投入后的费用。如某医疗保险信息系统，它是由高档数据库系统(Informix)和事务系统(TreeSvr)组成，在完全依赖高档数据库做数据处理时，效果十分不理想，而采用有事务能力的数据库系统（取其处理）与高档数据库系统（取其数据管理、分布能力）合作开发的具体应用系统，效果要好得多。以前用户受限制的那些表格都可以轻松处理。

        事务系统具有日常业务的支持能，如进销存系统中的采购计划、入库、出库。

  3. 存取界面

        是向信息系统获取信息的人机交互界面，可以表现为专门编制的程序、济览器里的页面、语音提示系统、多媒体模拟现实系统等。

  4. 历史数据采集（知识积累）

        数据仓库(Data Warehouse)、数据集市(Data Mart)概念就是基于知识积累而提出来的，它们不再是简单的数据库存储和管理，它们是一种解决方案是对原始数据进行各种处理并转换为有用信息的处理过程，这们为用户提供了一定的数据智能性。

        目前投入使用的绝大多数信息系统都只是当前（或某一时刻）静态数据的管理，而对被修改了的数据则缺少一种保存机制（注意这不同于一般意义上的备份）。如人事系统中只管了现员的当立脚点状态，而人的职称、履历、工资调整、不同时期人员构成等则缺少有效机制予以保存。数据的规律的总结形成知识，而要想在变动着的数据中寻求规律，则首先应对数据有个合理的抽取和保存。

  5. 辅助分析界面

        提供给决策者的决策支持系统（DSS）。

        信息系统的灵魂是贯川于其始终的数据处理语言和支撑运行平台。数据库系统和事系统是这个处理语言的载体，由它们组成了一个软件平台。信息系统，尤其是面向生产的信息系统，就靠这个平台运行。数据存取界面、历史数据采集、辅助分析界面则利用这个处理语言完成自己特定的功能，当然，在这三个系统内部可能还有更细的信息的处理语言。

        比如若存取界面支持自然语言，则它还有更多的自己的机制。

        在讨论数据处理语言之前有必要首先弄清信息系统的工作领域。用户寻找感兴趣数据的方便性及操纵数据的能力决定了信息系统的生产力，随着时间的变化，用户的需求会变化，但主要还是集中于几个方面：

    1. 特定存取

          用户按照应用系统提供的数据定义，利用应用系统功能或一些工具对数据作定存取和操作；

    2. 关系存取

          基于关系数据模型的数据的存取;

    3. 文本存取

          通过对文本的检索，找出用户感兴趣的存于各类文本编辑器存下的文本文件；

    4. 超媒体存取

          数据被分解成称作节点的单元，节点之间通过超连接相互连接，用户通过交互或运算等形式依不同路径对节点上的信息进行访问；

    5. 模糊存取

          能捕捉不确定的数据；

    6. 信息提取

          能提取散乱在各处的信息；

    7. 信息处理

          能对信息经一定处理后提取。

    不难总结，数据处理语言应具备的能力：

    1. 充分的信息表达能力

          信息系统中的任务描述起来有接近于自然语言般复杂，将如此复杂的信息予以形式化是实现信息系统一定要涉及到的工作，设计一个包罗万象的形式语言是不可能的，但设计一个当前系统适用的语言是必须的。

    2. 描述的任务要明确，不可出现二义性，客户和服务器有时可能运行在完全不同的环境中，由不同人来开发，出现二义性将使以后的维护及扩充变得异常困难。这里的二义性包括字节顺序、文字编码等方面。

    3. 务粒度要合适，以客观事物处理的自然方式为依据划分客户方与服务器方的任务，该任务的大小为自然事物中最小事件加上一定扩充。

    4. 交换信息描述要考虑到信息的传递，一般描述的信息应可集合到一个文件中。

    ASQL就是一个旨在完成这一使命的数据处理ViTAB和TreeSvr则是其单机和服务器版的运行平台。 因为ASQL是参照SQL而定义的，所以称为ASQL。
